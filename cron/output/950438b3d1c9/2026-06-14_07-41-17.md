### 1. Direct Preference Optimization for Chatbot Fine-Tuning: An Empirical Study
**Submitted:** 2026-06-11

### 2. ALIGNBEAM: Inference-Time Alignment Transfer via Cross-Vocabulary Logit Mixing
**Submitted:** 2026-06-10
**Link:** [https://arxiv.org/abs/2606.12342](https://arxiv.org/abs/2606.12342)

Introduces a training-free method to transfer safety alignment between model families at inference time by translating anchor logits into the target model's vocabulary token-by-token. ALIGNBEAM substantially raises refusal rates on adversarial benchmarks while keeping task accuracy and inference overhead within practical bounds — no weights are changed.

---

### 3. Which LoRA? An Empirical Study on the Effectiveness of LoRA Techniques During Multilingual Instruction Tuning
**Submitted:** 2026-06-09
**Link:** [https://arxiv.org/abs/2606.10428](https://arxiv.org/abs/2606.10428)

Investigates whether LoRA variants outperform basic LoRA in multilingual instruction tuning. Findings show no significant advantage from more complex LoRA variants — hidden embedding analysis reveals that layer-wise language representation remains largely similar regardless of the LoRA technique used, suggesting architectural novelty may not translate to better cross-lingual adaptation.

### 4. How Far Can Prompting Go for Minimal-Edit Ukrainian Grammatical Error Correction?
**Submitted:** 2026-06-08
**Link:** [https://arxiv.org/abs/2606.09334](https://arxiv.org/abs/2606.09334)

Evaluates 11 commercial LLMs and one open-source Ukrainian model on grammatical error correction, comparing zero-shot, few-shot, minimal-edits, and LLM-assisted prompt optimization strategies. The best configuration (Gemini 3.1-Pro) closes over 90% of the gap to fine-tuned SOTA (F0.5=69.22 vs 73.14), showing that sophisticated prompting can rival fine-tuned models for this task.

### 5. PrivCode++: Latent-Conditioned Differentially Private Code Generation for Comprehensive Guarantees
**Submitted:** 2026-06-08
**Link:** [https://arxiv.org/abs/2606.09145](https://arxiv.org/abs/2606.09145)

Addresses the challenge of differentially private fine-tuning for code generation where both prompts and code snippets are sensitive. Introduces PrivCode-Plus, a two-stage DP framework with a Privacy-Free Latent Conditioning module that enables effective DP fine-tuning and data synthesis without direct access to sensitive data — achieving substantially higher utility than DP baselines while providing stronger privacy guarantees.
